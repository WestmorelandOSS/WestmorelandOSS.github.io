Oct 6th 2015: Git To the Future

Teaching Git is hard! (almost as hard as it is to learn)

Git is a big deal and I want you all to learn it too. Deep down I think Git is pretty easy if you see it as a pattern of buttons to click in order. But it can get messy fast when you are working with others.

My goal is not to teach you how to fix all the kinds of conflicts that you can generate with Git. I will show you the best practices when working with Git so you can avoid most of them.

We will cover:
  • Repositories and their histories
  • Why you should trust Git
  • Branching and Merging
  • Collaborating and Merging conflicts (with tools that make life easier)
  • The mighty rebase (Love it and Fear it)

To register for this event please join WOSS

Agenda:
  • 6:00pm - Install Party for Git and Lightning talks
  • 7:00pm - Presentation on git and time to work on examples
  • 9:00pm - Questions discussion

Oct 15th 2015: Data Visualizations with Dave Columbus

Dave Columbus will take us on an adventure of graphic proportions. He has spent years collecting and organizing data about pop culture and creating visual arts. His visualizations express truths and debunk common myths about our favorite television shows and movies.

Dave will give us a survey of how data visualizations are constructed and the many different ways data can be displayed. This will cover generating static non-interactive works and his current exploration of dynamic interactive visualizations.

It is going to be great fun. The format will be a presentation and then QA / group driven discussion.

Dave on Twitter @pearvert

  • 7:00pm - QA \ Group Discussion about Data Visualization

Oct 29th 2015: Dynamic Websites and Web Services with Ruby Sinatra

Last time we learned how to build static websites with Ruby Middleman but what if we wanted to create web sites and services that could respond to the context of user requests. We will explore dynamic web pages and how to provide data to these page. How to react to data provided by the user and how the visualize the request response cycle between servers and browsers. This will open the door a little further towards learning some simple Ruby syntax and some of the core mechanics that more complicated frameworks like Ruby on Rails relies on.

The structure will be an initial lecture/presentation to familiarize you with some of the terminology in the middleman documentation. Followed by a learning circle with a pre-prepared project to help you get your feet wet with Middleman. A learning circle is where we will work with each other to build the sample project or your own project. There will be no defined proctor if you learn something you can share it openly. There will be experts present so you shouldn't have to worry about getting stuck.

Here is what you need to bring with you:
  • Laptop with Ruby installed
  • Your smiling face and open mind
  • (Optional) A simple web service idea
